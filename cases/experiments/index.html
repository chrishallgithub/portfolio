A Hard Days Night - WebRTC Experiment
Background

WebRTC (Web Real-Time Communication) is a new technology that will be introduced to web browsers in the coming months. It allows video chat and voice calling without the the need of plugins. As the stream is browser-based, it is possible to manipulate sound and video in real-time with the use of Javascript.
Solution

I made a WebRTC webapp to familiarise myself with how the new technology works. It essentially makes the user the star of The Beatles' A Hard Day's Night album cover. It captures a user's image every few seconds (allowing them to strike a different pose), and overlays this image on one of the windows of the album cover. WebRTC is so cutting edge that it only works on the most recent version of Google's Chrome Browser. Other browsers will be supported in the future.

How To Use

Just click: A Hard Day's Night, allow access to your webcam, and strike pose.
Send links from Browser to Mobile Device using QR codes
Background

I was browsing the internet on my computer one day and realised that there wasn’t a way for me send the link of the site I was looking at to my phone in a quick way. There are apps, such as Chrome To Phone, but my Android was version 2.2, which did not support the Chrome browser.

Solution

I wanted to make a quick way to send links to my phone that was as simple as possible. I decided to use a combination of QR codes, Google’s Chart API and a small Javascript bookmarklet. Google have recently decided to depreciate the use of chart images in their API though, so there is the possibility that this will not work.

To send a link to my phone, I click the bookmarklet on the page I am browsing. That sends the link to Google’s Chart API, which converts the link to a QR code. I then scan the QR code with a QR code reader on my phone, et voila, I’m looking at the site on my mobile.

It might sound complicated, but it is actually only three clicks until the page is viewed the mobile device.

Here’s the javascript bookmarklet:

javascript:(function(){var page=window.location.href;window.location="https://chart.googleapis.com/chart?chs=250x250&cht=qr&chl="+page;})();

What does that mean?

"javascript:(function(){" is just a way to tell the browser that you want to execute a Javascript function.

“var page=window.location.href;” creates a variable called “page”, and sets this variable to be the page you are currently looking at.

“window.location="https://chart.googleapis.com/chart?chs=250x250&cht=qr&chl="+page;" then redirects the page to go to Google’s Chart API service, and tells the service to create a QR code out of the “page” variable that was set previously.

How To Use

To create a bookmarklet, make sure the bookmark bar is showing on your browser. Right click any empty space on the bookmark bar, then create a new bookmark. Call the bookmark whatever you like (Awesome QR Creator, perhaps?), but copy and paste the full Javascript snippet into the URL location. Make sure there are no spaces.

Alternatively, just select the text of the full bookmarklet, and then drag it to your bookmark bar on your browser.

Have the bookmarklet show on your browser bar, and then click on it whenever you want to create a QR code version of the page you are visiting.
